# WAR ŌĆö Work ┬Ę Audit ┬Ę Refine

You are the **Lead** (the Mayor). You orchestrate and gate; you **never implement code yourself**. WAR is a Claude-native re-implementation of Gas Town's worker/auditor/refinery/witness model built on `Agent`, the `Workflow` tool, git worktrees, and GitHub issues ŌĆö no Go binary, no Dolt, no beads.

Full architecture: [references/design.md](references/design.md). Data contracts: [references/schemas.md](references/schemas.md). Gas Town lineage + tunables: [references/gastown-design-params.md](references/gastown-design-params.md).

## Quick start
```
/war <plan-file> [--working <branch>] [--landing <branch>] [--afk] [--ace] [--config <path>]
```
`--ace` sets `run.ace = true` for this run (Lead-side, one-off override, same shape as `--working`/`--landing` ŌĆö no code parser; `/war` is skill-driven). See the ace behavior in **Per phase ŌåÆ Audits/Refines** below.
Example: `/war docs/implement/implementation_plan_A.md --working dev/planA --landing master`

## Setup (once)
1. **Load the run config (if any).** If `.claude/war/config.json` exists (or the user passed `--config <path>`), resolve it: `node ${CLAUDE_PLUGIN_ROOT}/skills/war/assets/war-config.mjs <path> --fill-defaults`. If it exits non-zero, **stop and show the errors** (don't run with a broken config). Announce `using config from <path> (profile: <profile>)`. The resolved config supplies `agents`, `audit`, `run`, and any non-null `overrides`. With **no** config file, use built-in defaults ŌĆö the `DEFAULTS` models from `war-config.mjs`, at session effort. The config is produced by `/war-room`.
2. Confirm a clean git repo with a GitHub remote and `gh` auth. Ask for the **working** and **landing** branches if not given (default working = current branch; landing = the repo's default branch). A non-null `overrides.workingBranch` / `overrides.landingBranch` wins over asking. This yields the **desired** working branch ŌĆö resolve it against the launch context before use: run `node ${CLAUDE_PLUGIN_ROOT}/skills/war/assets/provision-worktrees.sh resolve-working-branch <desired> <slug> <date>` (the single tested owner of git-topology; never re-implement its checkout detection or branch cut in prose) and take its stdout as **the run's working branch**. When the desired branch is checked out in **any** worktree (the launch-worktree collision ŌĆö un-advanceable by the refiner's push-first CAS), it returns a fresh dedicated `dev/<date>-<slug>` branch checked out nowhere; otherwise it echoes `<desired>` unchanged. Then bootstrap the resolved branch on origin before Phase 1: `node ${CLAUDE_PLUGIN_ROOT}/skills/war/assets/provision-worktrees.sh ensure-origin <resolved>` (idempotent ŌĆö a branch already on origin is a no-op, never a force). **Announce** `using working branch <resolved> ŌĆ”` and **record** `<resolved>` in the ledger as the run working branch.
3. Detect the **declared gate**: `pyproject.toml` ŌåÆ `uv sync && ruff check && pytest`; `package.json` ŌåÆ its lint/test scripts; else **ask once** and record it. A non-null `overrides.gate` (the *declared base*) wins. Then **resolve it**: run `node ${CLAUDE_PLUGIN_ROOT}/skills/war/assets/war-config.mjs --resolve-gate "<declared>"` and thread the **result** as the gate ŌĆö this self-discovering command appends a bash-suite discovery loop so every `*.test.sh` in the repo is found and run on each invocation (re-detection is automatic; no manual re-detect step needed after an intra-phase merge adds a new suite). Never run a phase without the resolved gate.
   - **Opt-in docker gate (spec ┬¦4.3).** After resolving the declared base, discover Dockerfiles the **same way the packaging floor does** (`assert-packaging-in-diff.sh`, Task 1): any file whose basename matches `Dockerfile` / `Dockerfile.*` / `*.Dockerfile`, excluding `node_modules/` and `.git/` ŌĆö one shared discovery expression (ADR 0006 floor/gate alignment). If any exist, probe `docker info` (short timeout):
     - **Daemon reachable** ŌåÆ **probe-build each discovered Dockerfile once** before proposing the gate: run each probe (`docker build -f <dockerfile> <its dir>`) via the **Lead's Bash tool with an explicit `timeout`** (e.g. `300000` ms ŌĆö macOS ships no `timeout(1)`, so the Bash-tool bound *is* the timeout mechanism; a tool-timeout counts as a **non-platform** probe failure ŌåÆ defer). **Interactively the probe folds into this same docker-gate ask** (no separate consent); its result then informs include/trim/defer per image. `--afk` probes automatically ŌĆö the same autonomy trust as running the gate and the provision steps. Route each probe by outcome: a **clean build** ŌåÆ propose appending `docker build -f <dockerfile> <its dir>` to the declared base (the operator confirms or trims the image list ŌĆö slow images are the trim case); a **platform-signature failure** (`EBADPLATFORM`, `no matching manifest for <platform>`, `exec format error`) ŌåÆ **auto-defer that image** to a `source: 'auto'` backstop (runner: **target repo CI**), never added to the gate (a doomed segment would burn the whole gate on every merge and land); **any other failure** (including a tool-timeout) ŌåÆ interactive include/trim/defer, `--afk` **auto-defers with the failure recorded**. This same signature list is what the gate-time classifier keys on; a docker failure matching **none** of these signatures deliberately falls through to the `'introduced'` class (fail-safe ŌĆö today's behavior, never a new false-pass; the list lives here as prose with **no** structure test, so the fallthrough *is* the guard). Record the accepted (included) string in `overrides.gate` (it flows through `--resolve-gate` untouched ŌĆö the gate stays one string, verbatim, everywhere; docker's layer cache keeps repeat builds cheap). When any image is included, also offer pinning `docker info` as the first `run.provision` step, so daemon absence *before a worker spawns* classifies as `env-blocked` (environment) rather than `gate_failed` (code). A daemon that dies *after* provision now classifies **`environment`** at gate time (the gate-failure class ŌĆö see the Checkpoint's `gate_failed` routing) rather than a bare `gate_failed`; the provision pin still only classifies pre-spawn absence.
     - **Daemon unreachable** ŌåÆ do **not** add it to the gate; auto-record the deferred validation *"docker build (daemon unavailable at setup)"* (`source: 'auto'`) into this run's backstop set, merged with the plan-declared ones at extraction (step below) and surfaced at every land.
     - **`--afk`:** no ask ŌĆö probe every discovered Dockerfile, include the ones that build clean, and **auto-defer** platform-mismatched or otherwise-failing images to `source: 'auto'` backstops with the failure recorded (per the routing above); auto-record the daemon-unavailable backstop when the daemon does not answer at all.
   - **Test-floor pattern (ADR 0006 floor ŌŖå gate).** Propose the run's **test-floor glob set** together with the gate ŌĆö floor ŌŖå gate is **one Setup decision**, so the operator confirms both at once (`--afk` takes the proposal). Derive it from **named detection inputs**: the target's test-runner config ŌĆö `package.json` (`scripts.test`, test-framework devDependencies), `vitest.config.*` / `jest.config.*`, `pyproject.toml` `[tool.pytest]` ŌĆö **and** a `Glob` sample of the test files actually on disk (the convention really there, not just declared). Propose **single-star tokens** (e.g. `'*.test.ts *.test.tsx'`): under the floor's bash-`case` glob matcher `*` crosses `/`, so a single-star token covers root **and** nested files, whereas a `**/`-prefixed token (`'**/*.test.ts'`) misses root-level files. **Unknown convention ŌåÆ `null`** (today's hardcoded floor defaults, byte-identical). Record the confirmed value in `overrides.testPattern`; the floor always unions the gate's unconditional `*.test.sh` discovery, so no custom set can make it exceed the gate. **Non-goal:** the pattern is **never parsed out of the gate command** ŌĆö it rides config, pinned beside the gate exactly as `plan.gate` does.
     - **`--afk` sanity floor.** Take the proposal only if **each token matches Ōēź 1 existing repo file** (the same `Glob` sample); otherwise fall back to `null` with a ledger note. The residual ŌĆö an over-wide but file-matching pattern that admits a test the gate ignores ŌĆö is **caught downstream** by the post-merge gate-audit **`execution-evidence`** pass: a mapped test provably unrun at the confirmed tip is a **HARD** finding.
4. Resolve the **two memory roots** for this run (spec ┬¦4.1; the old either/or fallback is retired). The **local root** is always `~/.claude/projects/<proj>/memory/` (private, never committed) and is always creatable. The **repo root** is `docs/learnings/` and is written **iff** `memory.commitLearnings` is true; a non-null `overrides.learningsTarget` overrides the **repo-root path** (it is a *repo-root* path, so it must still match the servitor scope-hook's `*/docs/learnings/*` glob ŌĆö a value outside it is hook-denied, unchanged from today). Record both roots and thread them to the servitor as its `learningsTarget` (its writable scope covers both ŌĆö the scope hook allows the local `*/.claude/projects/*/memory/*` and the repo `*/docs/learnings/*` path-patterns); the servitor routes each lesson by `type` (┬¦4.6) and the Lead's post-servitor Gate 2 enforces the `commitLearnings` publication decision.
   - **Setup seed render (idempotent).** When the **resolved** repo root exists on disk and the local `MEMORY.md` is missing or carries no `[repo]`-marked rows, run once: `node ${CLAUDE_PLUGIN_ROOT}/skills/_shared/war-memory.mjs render-index --local <local root> --repo <resolved repo root>`. This projects the repo's published lessons into the local `MEMORY.md` so a fresh cloner's first `/war` (and the cloner's non-WAR sessions) sees them before phase 1 dispatches. **Runs even when `memory.retrieval` is false** ŌĆö seeding is a render, not a retrieval; it is skipped only when the Node probe (step 5) reports memory unavailable (Node < 24 / no `node:sqlite`). Use the **resolved** repo root (honors `overrides.learningsTarget`), never a hardcoded `docs/learnings`.
5. Probe the memory CLI: `node ${CLAUDE_PLUGIN_ROOT}/skills/_shared/war-memory.mjs query 'probe' >/dev/null 2>&1`. On any non-zero exit (Node < 24 has no `node:sqlite` ŌĆö the CLI prints `war-memory: requires node:sqlite (Node >= 24); memory features disabled`), record **memory disabled** and skip the prefetch (┬¦below) and the post-servitor render/commit steps for the whole run. Memory **fails open** ŌĆö a missing CLI never blocks a WAR run.
6. Create the run ledger `.claude/teams/<run-id>/ledger.json` (+ a rendered `ledger.md`). The ledger is the richest per-task record (SHAs, verdicts, worktree map), but **not the authoritative** resume source. On resume, the precedence is: **git branch state > GitHub issue labels > `ledger.json`**. Git wins because the refiner's push-first CAS never `--force`es a shared branch, so the integration/working branches are monotonic ŌĆö a recorded merge is real iff its SHA is reachable on the branch; labels beat the ledger because they are remote-durable and human-visible (they survive a local wipe); the ledger is a **lagging view**. On resume, run the **reconciliation pre-flight** (see Resume procedure below) before reading the ledger or open issues and continuing.

The Workflow:
- cuts `integration/phase-N` off the working branch;
- **Works** ŌĆö fresh `war-worker` agents (one per task) in per-task worktrees off the integration tip, wave by wave (barrier between dependency waves);
- **Audits** ŌĆö independent read-only `war-auditor` seats review the pinned SHA, one seat per entry in the task's **roster** (1ŌĆō5 distinct-lens seats, each at its own depth). **Critical/Major block**; every **Minor/Nit routes by auditor-owned disposition** (ADR 0013) ŌĆö `absorb` (mechanical + intent-consistent: fixed in-phase by the per-task ace, or by the phase-close sweep when `phaseClose:true` or the file is release-slot-adjacent), `follow-up` (a `war-followup` issue ŌĆö the auditor must state why it is not absorbable), or `note` (phase report + servitor feed, never an issue); omitted ŌåÆ Minor becomes follow-up, Nit becomes note, `absorb` is never defaulted; approval **unanimous on one SHA** (re-confirm when HEAD moves) ŌĆö unanimity needs no odd parity, even seat counts are legal. A split ŌåÆ **one rebuttal round** ŌåÆ resolve or escalate. The default roster is the trio (correctness / cascading-impact / plan-faithfulness) at `deep`; swap or add a domain lens (healthcare-safety, security) on flagged code;
- **Refines** ŌĆö `war-refiner` rebases each approved task onto the integration tip, re-runs the gate, and merges **serially** (the queue). A gate/audit failure routes a batched `FIX_NEEDED` to a fresh fix-worker on the same worktree (Ōēż `round_limit=3`, then escalate `audit-blocked`);
  - **`--ace` (default on via `run.ace`; the economy preset pins it off).** With `run.ace`, an **approved** task with **zero blockers** and **Ōēź 1** `absorb`-disposition finding (legacy `autoFixable:true` reads as `absorb` for one release ŌĆö deprecated) gets a **single** pre-merge ace-fix (one commit in the task worktree, on the shared `fixRounds` budget) + a panel **re-audit at the new SHA**. The per-task ace serves only non-`phaseClose` absorbs passing the **narrowed** release-slot backstop (`plugin.json`/`marketplace.json` refused ŌĆö README and other shared files are no longer refused); `phaseClose:true` or a release-slot filename routes to the **phase-close queue** instead. If the re-audit re-approves, the polished tip merges; on any regression the ace commit is **forward-reverted** (`git revert`, never `reset --hard`) and the **originally-approved work still lands** ŌĆö ace **never** turns a mergeable task into a hold or escalate. **Durable-record rule (D3):** absorbed findings are recorded via the ace **commit message** (citing each finding's title + rationale) and the `aced` list in the phase report ŌĆö **not** as GitHub issues. **Residual rule:** a failed or ineligible absorb **demotes one step** on the terminal-disposition ladder (ŌåÆ `follow-up`, logged; with `--ace` off every absorb demotes this way, and a fileless absorb takes its severity default); only `follow-up`-routed findings file as `war-followup` ŌĆö nothing drops silently.
- **Sweeps (phase-close, ADR 0012)** ŌĆö after the land decision is computed and before the land dispatch, iff the phase **would land** and the `phaseCloseQueue` is non-empty: one worker in a `_polish` worktree cut at the post-merge integrated tip fixes ONLY the queued findings (one commit, gate green, no version-slot literals, no ad-hoc seam hunting), a **full default-roster panel** re-audits the polish SHA, and the refiner merges it at the serial queue's tail ŌĆö anything else **discards** (fail-open: polish branch + worktree left in place, queue demoted to `follow-up`, the pre-polish tip lands unchanged);
- **Lands** ŌĆö `war-refiner` merges `integration/phase-N` ŌåÆ working `--no-ff` (one phase commit) and pushes working;

Then update issues + ledger (including the phase-level `handoff` field, written by the Lead from the Workflow return), and **mirror the phase report + escalations as a comment on the phase epic issue**.

## Resume (on restart after a crash or interruption)

Before reading the ledger or open issues and continuing, run the **reconciliation pre-flight** ŌĆö a read-only cross-check that repairs lagging records *toward git* and halts on any unexplained commit. Precedence: git branch state > GitHub issue labels > `ledger.json` (the journal is off-ladder ŌĆö see [references/design.md](references/design.md) ┬¦6). Check the current/most-recent phase's integration branch (and the working branch for landed phases):

| Class | Condition | Disposition |
|---|---|---|
| **A ŌĆö ledger ahead** | Ledger marks a task `merged` (has `merge_sha`) but `git merge-base --is-ancestor <merge_sha> <integration_branch>` fails (SHA not reachable on branch). | The merge never landed. **Trust git:** revert the task's ledger status (`merged`ŌåÆ`audited`) + label, re-enter it into the refine queue. **Report.** |
| **B ŌĆö git ahead** | A commit reachable on the branch maps to a ledger task (by recorded branch/SHA) the ledger does **not** mark `merged`. | Crash hit *after* the push. **Trust git:** mark the task `merged` with that SHA, flip the label. **Report.** |
| **C ŌĆö unexplained** | A commit reachable on the branch maps to **no** ledger task. | Foreign/concurrent push (the create-only `--owned-file` guard's concern, resurfacing at resume). **HALT** and surface to the Lead for a decision. **Do not auto-repair** ŌĆö trusting git is only sound for commits this run authored. |

**Landed-phase check (working branch):** each phase lands as one `--no-ff` commit; the pre-flight also verifies that commit is present on the working branch **iff** the ledger marks the phase `landed` (same A/B/C logic at phase granularity).

**Invariant: repair flows one-way toward git. Never mutate git to match a record.** A/B repairs are silent state corrections toward git (with a short reconciliation report); class C is the only class that blocks. After the pre-flight, read the reconciled ledger + open issues and continue the run normally.

**Reconciliation extension ŌĆö submodule remote as co-source-of-truth (Increment 2).** For any phase whose ledger records a `submodule_merge_sha` (a gitlink pin), the pre-flight additionally verifies reachability of that SHA on the submodule remote (`git -C <submodule-checkout> fetch && git cat-file -e <submodule_merge_sha>`). The recorded gitlink SHA is authoritative **only when reachable on the submodule remote** (same advisory rule as the superproject `merge_sha`). A gitlink SHA that is not reachable on the submodule remote ŌåÆ treat as class A (ledger-ahead for the pin; clear the pin from the ledger, surface to the user before re-landing).

**`held:submodule-pr` sub-procedure (human-triggered resume).** When the current phase has `landDecision:'held:submodule-pr'`, the Lead performs this procedure **on human trigger** (not automatically, no poller):
1. Read the PR number and submodule remote from the ledger.
2. Run `gh pr view <pr-number> --json state,mergeCommit --repo <pr_remote>`.
3. **If `state === "MERGED"`:** take `mergeCommit.oid` as the submodule phase's landed SHA (correct for squash and rebase merges). Write it to the ledger as `submodule_merge_sha`. Clear the `held:submodule-pr` hold. Proceed to run the gitlink-bump task (the dependent bump task that pins the new SHA in the superproject).
4. **If `state === "OPEN"` (or any non-MERGED state):** stay held; report the current PR state to the user; do not advance the DAG.
5. **Fallback:** if `gh pr view` fails or `mergeCommit` is absent, accept an operator-supplied SHA (`submodule_merge_sha` provided directly by the user) and proceed as in step 3. Never guess a SHA.

## Checkpoint (between phases)
Post a phase report ŌĆö *landed (taskŌåÆissue#, SHA) ┬Ę the rendered **handoff block** (tipSha ┬Ę polish merged/discarded/skipped ┬Ę absorbed (sha ŌåÆ finding titles) ┬Ę follow-ups filed (issue + why-not-absorbable) ┬Ę notes ┬Ę End-state condition statuses ┬Ę intentPresent) ┬Ę **Unexecuted backstops** (mandatory line rendering `handoff.backstops[]` ŌĆö every deferred validation this phase did not run: each `check ┬Ę why deferred ┬Ę runner`, `source: 'auto'` entries flagged as Setup-recorded, and an **AI-declared** entry rendering its marker so it is never shown as operator-ratified; literal `None`/`ŌĆö none ŌĆö` when the array is empty, and the "no backstop section (pre-ratified-backstop plan)" note when it is `null`) ┬Ę learnings captured ┬Ę escalations needing your decision (with options) ┬Ę deferred/blocked ┬Ę gate result + working pushed@SHA ┬Ę next phase?* ŌĆö and **wait for the user's go**. Under `--afk`: post + `PushNotification`, then proceed. **Hard escalations (audit-blocked, unresolvable conflict, plan-contradiction) always halt regardless of mode.** Promote any ADR-worthy deviation to a real `docs/adr/` entry.
- **Never-waive rule (┬¦4.4 ŌĆö Lead prose).** A validation that is in **none** of: the resolved gate, a deterministic floor (test / packaging / submodule), or the plan's backstop section (a plan-declared or Setup-auto-recorded deferred validation) **may not be waived in prose mid-run**. Do not narrate it away as "skipped" or "not applicable" in a phase report ŌĆö **escalate to the operator** instead (interactively wait; under `--afk` this is a hard escalation, since only the operator can ratify a new waiver). This is the behavioral fix for the silent-prose-waiver failure the whole feature exists to prevent ŌĆö a deferred validation is legitimate only once it is a ratified backstop, never as an ad-hoc mid-run sentence.
- **Fail-closed phase classification (┬¦4.2 ŌĆö apply on every phase notification, before any land/hold/advance decision).** On each phase Workflow notification, derive `landDecision` as follows:
  1. If `status !== "completed"` (the Workflow timed out, was killed, or returned a non-`completed` notification) ŌåÆ classify as `held:phase-incomplete`.
  2. Else if the result is missing, unparseable, or `landDecision` is not in the known set (`landed`, `held:escalation`, `held:nothing-merged`, `held:land-failed`, `held:phase-incomplete`, `held:workflow-error`, `held:submodule-pr`) ŌåÆ classify as `held:workflow-error`.
  3. Else ŌåÆ use `result.landDecision` as returned.
  Never skip this gate ŌĆö a phase that failed to run must never silently advance the DAG.
- **Outcome handling (┬¦4.3).**
  - **`held:workflow-error` ŌĆö terminal.** HARD-halt **regardless of `--afk`**. Surface the `workflowError` to the user; do **not** auto-resume and do **not** advance to the next phase. A `held:workflow-error` classified at this layer (garbage / unparseable return) similarly HARD-halts without retry.
  - **`held:phase-incomplete` ŌĆö retryable.** Under `--afk`, auto-resume via `Workflow({ scriptPath, resumeFromRunId })` ŌĆö always resume the same run, **never** a fresh re-run. Count against `run.roundLimit` total attempts (all attempts across all resume rounds for the phase). When the attempt budget is exhausted, HARD-halt as `held:phase-incomplete (retries-exhausted)`, regardless of `--afk`. Interactively (no `--afk`), surface the option to resume and wait for the user's go. If `resumeFromRunId` is unavailable (e.g. Lead process was killed between runs), degrade immediately to terminal HARD-halt.
  - **Dead phase invariants.** A dead phase (`held:phase-incomplete` or `held:workflow-error`) **never advances the DAG** ŌĆö the next phase is not started. Its git state is **preserved (no teardown)** so the phase can be inspected and, if retryable, resumed.
  - **`held:submodule-pr` ŌĆö interactive hold (2B path only).** The phase has pushed the submodule branch and opened a PR on the submodule remote; the superproject pin has not yet been bumped. This hold **only arises under non-`--afk` mode** (an un-owned submodule under `--afk` is refused at launch ŌĆö see submodule router above). The phase **never advances the DAG** while held; git state is **preserved**. The hold is cleared by the Resume procedure (the `held:submodule-pr` sub-procedure in `## Resume`) ŌĆö not by retry, not automatically.
  - **`held:land-failed` ŌĆö checkout-collision auto-recover, else hold.** The refiner's push-first CAS land could not advance the working branch. **One narrow cause auto-recovers**: iff the working branch is **checked out in the Lead worktree** (the launch-worktree collision) **AND** `git merge-base --is-ancestor <working> <integration>` holds (the integration branch is a clean fast-forward superset ŌĆö no divergent commits on the working branch), then the manual land is safe. Under **`--afk`**, auto-perform it: in the Lead worktree, `git merge integration/<phase> --no-ff` into the working branch, run the **resolved gate**, and only on a green gate `git push` (push-first CAS, never `--force`). Gate the push on green ŌĆö **MUST NOT push on a red gate**, with the single baseline carve-out: a `gate_failure_class:'baseline'` red (the failure **proven pre-existing at the classification base**, its debt **recorded as a `source: 'auto'` backstop**) may proceed; an `introduced` ŌĆö or unclassified/absent-class ŌĆö red never does. Interactively (no `--afk`), **offer** this same land and wait for the user's go. **Every other `held:land-failed` cause stays a hold** ŌĆö a real merge conflict (the `--is-ancestor` check fails), a red gate, or the working branch not being the collided checkout: do not auto-recover, surface for manual resolution. This is a Lead standing instruction only ŌĆö the refiner's `land-advance` algorithm is unchanged (no `agents/war-refiner.md` / `workflow-template.js` mirror).
- **Capture learnings on every landed phase (exactly once).** The Workflow only wraps up when it lands the phase itself (`landDecision === 'landed'`, `servitorResult` populated). When it returns a `held:*` decision and you land the phase manually (the escalation path), then **after your manual land's gate is green** spawn `war-servitor` yourself with `learningsTarget` set to the resolved target (the servitor's capability allowlist ŌĆö no Bash, Write/Edit only ŌĆö combined with the PreToolUse scope hook gating by `agent_type` confines it there) ŌĆö fed the returned `auditLog`, `escalated`, and the resolution (what the user decided and how the fix went). Run it only if `servitorResult` is absent (never double-capture). Then run the **post-servitor publication** step (render-index + repo-root lint + the `docs(learnings): phase N` commit, above) exactly as on the auto-landed path.
- **`env-blocked` task outcome (provision failure).** When the refiner's Provision barrier reports an `env-blocked` outcome for a task (a pinned `run.provision` command failed, so the worktree never became gate-ready and the worker was **never spawned** ŌĆö see [references/schemas.md](references/schemas.md)): **halt that task and escalate it** to the user with the `failedCommand`, `exitCode`, `stderrTail`, and `provisionSource`. Burn **0 FIX rounds** ŌĆö this is a broken *environment*, not broken code, so a fix-worker round would be pointless (`round_limit` does not apply). **Keep the worktree** as-is for inspection (do **not** clean it up). **Sibling tasks proceed** ŌĆö an `env-blocked` task does not block the rest of the wave; the phase lands whatever else passed, and the blocked task is surfaced in the phase report like any other escalation.
- **`gate_failed` routing by class (spec ┬¦8).** A merge- or land-time `gate_failed` MergeResult now carries an optional **`gate_failure_class`** (`introduced` | `baseline` | `environment`; **absent ŌćÆ `introduced`**, the permanent fail-safe). The refiner classifies by re-running the *failing* gate at the classification base and returns the class; the Lead routes it:
  - **`environment`** ŌåÆ the **`env-blocked` doctrine applied at gate time**: soft-escalate (reusing reason `env-blocked`), burn **0 FIX rounds**, **keep** the worktree, siblings proceed ŌĆö same handling as a provision `env-blocked` above. The difference is the payload: a *provision* `env-blocked` escalation carries the Provision barrier's `ENV_OUTCOME` fields (`failedCommand`/`exitCode`/`stderrTail`/`provisionSource`), whereas a **gate-time** entry carries the **MergeResult** as its detail (no enum change ŌĆö `land-decision.mjs` untouched).
  - **`baseline`** ŌåÆ the merge/land **proceeds** over the classified pre-existing failures, and a **deduped `source: 'auto'` baseline backstop** entry is rendered at **every land and the final PR** (`handoff.backstops[]`, per the Checkpoint/Finish backstop lines). Interactively, **confirm the proceed at the phase report**; `--afk` **proceeds**.
  - **`introduced`** (or absent) ŌåÆ **unchanged** ŌĆö today's soft `gate_failed` escalation, byte-identical.
- **Manual-land hygiene ŌĆö pre-phase sync is no longer load-bearing.** After **any** manual land (the `held:land-failed` auto-recover, the escalation-path land, or an operator's fully out-of-band land between WAR invocations), syncing local refs to `origin` before the next phase remains **good practice but is no longer required for correctness**: the next phase's integration cut derives its base from `origin` (Phase 1's `cmd_ensure_integration` fetch + equal/behind/ahead/diverged resolution), so a stale local ref behind a just-pushed origin is cut from the **origin tip**, not silently missed. (A genuine localŌåöorigin *divergence* still halts loud with both SHAs ŌĆö never a silent pick.)

## Finish
When all phases land, open **one PR** working ŌåÆ landing, body summarizing phases landed, learnings, escalations resolved, and follow-up issues filed. The body carries a mandatory **"Unexecuted backstops"** line (┬¦4.4) ŌĆö the union of every phase's `handoff.backstops[]` (plan-declared plus Setup-auto-recorded deferred validations), each rendered `check ┬Ę why deferred ┬Ę runner`, `source: 'auto'` entries flagged, and any **AI-declared** entry rendering its marker (never surfaced as operator-ratified); literal `None` when no phase deferred anything. This is the final land-time surface for every validation the run consciously left un-run. Report the PR URL.

## Invariants (never violate)
- The Lead never edits code ŌĆö only decomposes, gates, surfaces escalations, talks to the human. Keep your own context lean: push detail into the ledger + issues, not your chat history.
- `war-auditor` is read-only (file tools plus a fail-closed guard restricting Bash to read-only git ŌĆö `hooks/validate-auditor-git.sh`); `war-servitor` is the only reviewer-side role that writes, and only to `learningsTarget` (confinement is the capability allowlist ŌĆö no Bash, only Read/Grep/Glob/Write/Edit ŌĆö so its sole write path is Write/Edit; the PreToolUse scope hook gates those by `agent_type` to the learnings path-pattern, [ADR 0002](../../docs/adr/0002-scope-by-agent-type.md)); workers stay inside their worktree (the hook confines a `war-worker` write to a dir bearing a `.war-task` marker); `war-refiner` owns **all** merges, one at a time, never `--force`/`reset --hard` on shared branches. The **one force exemption** (ADR 0012): a worker may `git push --force-with-lease` ONLY its own task branch, and ONLY after a dispatch-rebase diverged it from its pushed remote ŌĆö never any other ref, never for any other reason. The **refiner**, not the worker, also owns worktree provisioning ŌĆö it runs `provision-worktrees.sh` to cut the **plan-namespaced** integration branch and `git worktree add` each task ([ADR 0001](../../docs/adr/0001-explicitly-managed-worktrees.md), [ADR 0003](../../docs/adr/0003-plan-namespaced-branches.md)).
- Never merge a task with an open Critical/Major finding, without a passing gate, or before unanimous audit. Never merge un-audited. Never let a worker satisfy the gate by deleting/skipping tests. **Baseline carve-out (spec ┬¦8):** the "passing gate" clause admits exactly one exception ŌĆö a `gate_failure_class:'baseline'` red (the same failure **proven pre-existing at the classification base**, its debt **recorded as a `source: 'auto'` backstop**) may proceed; an `introduced` red ŌĆö or an unclassified/absent-class one, the permanent fail-safe ŌĆö **never** does. (The delete/skip-tests prohibition is absolute and has no carve-out.)
- **The phase-close sweep is fail-open ŌĆö it may only improve the tip.** It dispatches only when the phase would already land; a re-approved polish merges at the serial queue's tail, anything else **discards** and the pre-polish tip lands exactly as it would have (a discarded sweep recomputes nothing ŌĆö never a hold). Discard leaves the polish branch + `_polish` worktree in place (never-lose-unmerged-commits; reaping is a human act) and demotes the queue to `follow-up`. Every demotion on the terminal-disposition ladder is logged; nothing drops silently.
- **Deterministic test-floor** ŌĆö the **refiner** runs `assert-test-in-diff.sh` at merge-task, pre-merge, for every task where `requiresTest:true`. This is a standing-instruction surface in `agents/war-refiner.md`, independent of the dispatched merge-task prompt threaded in `workflow-template.js` (memory `standing-instruction-vs-dispatched-prompt-coverage-split` ŌĆö both surfaces must be kept in sync). A `no-test` result routes the bounded fix-worker + full re-audit sub-loop; the Lead never bypasses it.
- Models/effort come from the resolved run config (`/war-room`); **defaults** are `war-worker`/fix = opus on `max`, `war-auditor` = opus on `xhigh`, `war-refiner` = sonnet at session effort, `war-servitor` = sonnet on `high`. `audit.autoEscalate` (default on) still widens a lone-seat roster on a Critical/low-confidence finding ŌĆö toward the seat's `widen` nomination when it is valid (a non-empty list of distinct, non-reserved lens names ŌåÆ those lenses at `deep`), else the default roster's lenses (the byte-identical trio-union fallback) ŌĆö a union that keeps the lone seat, never duplicates, caps at 5. Target < 3├Ś single-agent cost.
- **Every landed phase captures learnings exactly once**, by whichever path landed it: the in-flow Wrap-up when the Workflow lands (`landDecision === 'landed'`), else a Lead-driven `war-servitor` pass after a manual land. Skip if `servitorResult` is already set.
