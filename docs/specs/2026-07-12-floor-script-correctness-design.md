# Floor-script correctness: Dockerfile continuation join, ref-accurate .gitmodules, guard-message extraction fidelity, vacuous-packaging advisory

**Issues addressed:** #800, #802, #803, #819
**Date:** 2026-07-12 · **Author:** /survey-corps (Survey Corps spec pass)
**Depends on:** `docs/specs/2026-07-12-audit-gate-evidence-fidelity-design.md` — build **after** it. That spec reshapes the gate-audit evidence lane in `skills/war/assets/workflow-template.js`; this spec edits the same file's dispatched merge-prompt sites (#819 flag threading) and improves the guard-specificity evidence (#803) that lane consumes. Landing this first would force a re-merge of `workflow-template.js` and re-anchor the evidence prose twice.

## 1. Context — the gap / problem

Four recorded correctness ceilings in the refiner-side merge floors (`skills/war/assets/`). All are heuristic floors under the ADR 0006 posture (coarse floor, semantic ceiling, 0/1/2 exit contract); none currently corrupts a run, but each silently degrades what the floor proves.

**#800 — packaging-floor Dockerfile parser: continuations unjoined, O(N×M) re-parse.**
`parse_dockerfile` in `assert-packaging-in-diff.sh` tokenizes one physical line at a time, so a backslash-continued multi-line `COPY`/`ADD` (a very common Dockerfile style) collapses to too few tokens and its sources are silently dropped — the enumerated-COPY probe (`lit_dir_into`) then never fires and the floor fail-opens (exit 0, no flag) on exactly the Dockerfiles it exists to police. Separately, the main scan loop calls `parse_dockerfile "$D"` (a fresh `git show` + full re-tokenize) once per (added-file, Dockerfile) pair, though the parse is a pure function of `$branch:$D` — an O(N×M) waste. Both were deliberately Nit-graded at origin; the continuation gap is a real false-negative class, not just inefficiency.

**#802 — submodule floor reads working-tree `.gitmodules`, not a ref snapshot.**
Step 3 of `assert-no-submodule-mutation.sh` resolves declared submodule paths via `git config -f "$gitmodules_file"` — the *checked-out working tree's* `.gitmodules`, not `.gitmodules` as of the refs the diff actually spans (lesson `gitmodules-working-tree-read-vs-ref-snapshot`). Under the refuse-all default this is benign (the Step-2 mode-160000 gitlink check fires first), but under `--declared` mode — live-threaded from `workflow-template.js` for declared gitlink-bump tasks — a task branch that itself adds or edits `.gitmodules` is judged against a path set that silently diverges from the diff endpoints, and can be misclassified in either direction.

**#803 — guard-specificity floor: `extract_msg()` has two known mis-extraction shapes.**
`extract_msg()` in `assert-guard-specificity-in-diff.sh` returns the **first** quoted literal anywhere on an added line. Shape 1: a stderr-emit guard like `[ -f "$path" ] || { echo "real msg" >&2; exit 1; }` records `$path` as the guard message (and a `printf 'error: %s' "$x" >&2` guard records the uninterpolated format string), producing both false negatives (a truly uncovered guard masked by an accidental match on the wrong token) and false positives in the gate-audit evidence. Shape 2: a `die "$msg"` guard whose message is a bare quoted variable records the literal token `$msg`, which no test can ever substring-match — the floor perpetually flags it uncovered. The floor is advisory-evidence only (its header contract: exit 1 mints no MergeResult status and never blocks a merge), so the blast radius is gate-audit signal quality — the `guard_specificity` evidence recorded per merged task in the post-merge evidence dispatch — not run integrity. Neither shape is covered by `assert-guard-specificity-in-diff.test.sh`.

**#819 — packaging floor structurally vacuous with no advisory (SCOPED).**
`assert-packaging-in-diff.sh` exits 0 before per-file analysis when the branch tree has no Dockerfile, and its diff-collection arm keeps only `A*`/`R*|C*` paths, so a purely-Modified diff can never flag. **Both shapes are ratified as intended design** by the ADR 0017 addendum (2026-07-08), mirrored into the script header ("SCOPE RATIFICATION") — this spec does **not** overturn that. The sole residual is surfacing: when a plan *explicitly* declares `requiresPackaging: true` and the floor run is structurally vacuous, nothing tells anyone — `workflow-template.js` logs only the `requiresPackaging:false` skip. Since `requiresPackaging` defaults true, a naive always-on advisory would fire on every task in every non-docker repo; the advisory must key on explicit declaration.

## 2. Pivotal constraints

1. **ADR 0017 addendum stands.** The A/R/C-only diff-collection arm (`A*)` / `R*|C*)` / `*) continue`) and the trivial no-Dockerfile exit-0 are ratified byte-unchanged in behavior. #819 adds an *advisory line on stderr* behind an opt-in flag — exit codes, stdout, and scope untouched.
2. **The 0/1/2 exit contract is load-bearing** (mirrors `assert-test-in-diff.sh`): 1 = the named route (`unpackaged` / advisory-evidence `uncovered`), 2 = git error, never collapsed. No change in this spec may move any exit code, and stdout stays the machine-read surface (flagged `F -> D` pairs; uncovered-guard lines) — all new advisory output is stderr-only.
3. **Floor discovery patterns must mirror `resolveGate` in `war-config.mjs` and the gate find exclusions** — none of these changes may touch a discovery set. `assert-packaging-in-diff.sh`'s Dockerfile-naming discovery (`is_dockerfile`) and `assert-no-submodule-mutation.sh`'s `git diff --raw` inspection are exempt-by-domain (recorded in the 2026-07-11 drift-guard-tightening spec's classification); `assert-guard-specificity-in-diff.sh`'s byte-copied `match_sh_suite`/`match_default` gate-set discoverers are **out of bounds for this spec** — its edits stay inside `extract_msg`/`handle_added_line`/`record_guard`.
4. **Dispatched prompt sites change together.** The #819 flag threading edits every `workflow-template.js` prompt site that instructs running `assert-packaging-in-diff.sh` (initial merge, retry-merge, and the re-merge re-instruction site) in one commit; a partial thread is silent drift between rounds.
5. **bash 3.2.57-safe, cwd-independent** — no associative arrays, no globstar; test style mirrors the sibling `*.test.sh` suites. This rules out an eval-free per-path parse cache and motivates the loop-inversion resolution for #800.
6. **The guard-specificity floor stays advisory-evidence.** #803 is a fidelity fix to what the evidence *says*, never a promotion toward blocking; `HARD_ESCALATION_REASONS` and MergeResult statuses are untouched.
7. **Anchor by named construct** (`parse_dockerfile`, `extract_msg`, `handle_added_line`, the Step-3 comment banner), never line numbers — these files sit in the serial merge queue.
8. **Comments travel with rewrites** (lesson `source-comment-lags-emitted-prompt-after-rewrite`): every header-contract comment that states the old mechanism (the `git config -f .gitmodules` usage note, the `MSG = the first quoted string literal` contract, the "Uses D_COVERS from the most recent parse_dockerfile call" note) is updated in the same diff as its code.

## 3. Resolved design tree

| Decision | Resolution |
|---|---|
| #800 continuation handling: pre-join `\`-continued lines vs. reject/warn on continuations | **Pre-join.** Fold physical lines ending in a trailing backslash into one logical line before the keyword loop in `parse_dockerfile` (dropping full-line `#` comments encountered mid-continuation, which Dockerfile syntax permits). Rejecting would fail-open exactly as today; joining makes the enumerated-COPY probe see the real source list. |
| #800 re-parse elimination: eval-keyed global cache per Dockerfile vs. memo-last-parse vs. **loop inversion** | **Loop inversion.** Iterate Dockerfiles outer, added-files inner, parsing each Dockerfile exactly once. A per-path cache needs dynamic var names (eval) under bash 3.2; a last-parse memo is defeated by the current F-outer alternation. Inversion is the smallest correct change; the flagged pair *set* is unchanged, only line ordering may shift from F-major to D-major (test expectations updated in the same task). |
| #802 which ref supplies `.gitmodules` | **`$branch` (the tree being merged), via `$git_cmd config --blob "$branch:.gitmodules" --get-regexp '\.path$'`.** The floor judges the branch's content against the branch's own declarations; a base-side union would refuse legitimate de-submoduling (branch removes the entry + gitlink and adds regular content). The Step-2 gitlink-mode check remains the ref-independent backstop. |
| #802 absent-`.gitmodules` handling | **Existence probe first** (`$git_cmd cat-file -e "$branch:.gitmodules"`): absent → skip Step 3 clean (exactly today's no-file no-op); present but blob-read fails → `die … 2` (git error, never a clean pass). The working-tree fallback is removed, not kept as a fallback — a half-migrated read reintroduces the divergence. |
| #803 shape 1: which quoted literal is the message | **The first quoted literal *after the emitting token* and *before* `>&2`** — for `die`, after the `die ` token (already the case); for stderr-emit lines, after the `echo`/`printf` keyword rather than first-on-line. A `printf` format literal containing a `%` conversion is truncated to its literal prefix before the first `%` (a substring a real test assertion will contain); if the prefix trims to empty/whitespace it is treated as unmatchable (next row). |
| #803 shape 2: bare-`$var` messages | **Skip recording entirely** when the extracted token is a bare variable reference (`$name` / `${name}` and nothing else). Recording it flags the guard uncovered forever (no test can substring-match `$msg`); skipping degrades to "this guard is not checkable by this heuristic," the honest floor answer. Partially-interpolated literals (`"error: $x missing"`) still record — substring matching against the literal fragments still works. |
| #819 advisory placement: engine-side log vs. script-side stderr | **Script-side stderr, behind a new opt-in flag `--advise-vacuous`.** The Workflow sandbox cannot run git, so only the script knows Dockerfile discovery came up empty; the engine only knows whether the plan *explicitly* declared `requiresPackaging`. Split accordingly: the script owns detection, the engine owns the firing condition. |
| #819 firing condition | **Thread `--advise-vacuous` into the dispatched invocation only when `r.task.requiresPackaging === true` (strict equality — explicitly plan-declared, not the undefined→true default).** This is precisely the issue's anti-noise requirement: defaulted tasks in non-docker repos never see the advisory. |
| #819 which vacuous shapes advise | **Both ratified shapes**: (a) branch tree has no Dockerfile (the `dockerfiles`-empty exit); (b) non-empty diff with zero A/R/C paths (the `added_files`-empty exit when `diff_out` is non-empty). An entirely empty diff stays silent — nothing was packaged or not-packaged. Each advisory line names its shape and cites the ADR 0017 ratified scope, so the reader learns "vacuous by design," not "bug." |
| #819 evidence plumbing for the advisory | **None (YAGNI).** The advisory surfaces in the refiner's command output during the merge step; no MergeResult field, no schema change, no relay. If a future audit lens wants it, that is its own spec. |

## 4. Mechanics

### A. `parse_dockerfile` continuation join + parse-once inversion (`skills/war/assets/assert-packaging-in-diff.sh`, #800)

1. Inside `parse_dockerfile`, after `git show` captures `pd_body`, insert a join pass producing logical lines: a line whose last non-whitespace character is `\` is stripped of the backslash and concatenated (single-space-joined) with the next line; full-line `#` comments encountered while a continuation is open are dropped (Dockerfile-legal). The existing tokenizing loop then reads logical lines unchanged. `# ponytail:` comment naming the ceiling: a trailing backslash followed by trailing spaces is treated as no continuation (matching strict BuildKit behavior); heredoc/JSON forms remain deferred per the existing spec-§9 comment.
2. Invert the main scan loop: outer `while` over `$dockerfiles`, calling `parse_dockerfile "$D"` **once**; inner `while` over `$added_files` running the existing Step 1–4 probes against the parsed `D_LIT_DIRS`/`D_COVERS` globals. The per-F skips (`.dockerignore`, `is_dockerfile "$F"`) move inside the inner loop (or hoist to a pre-filtered `added_files` list — implementer's choice; behavior identical).
3. **Token sweep (grep is a floor):** grep `parse_dockerfile` across `assert-packaging-in-diff.sh` and rework every call/comment site for the parse-once shape — then **hand-scan the script's same-scope section banners and helper comments** (the "Uses D_COVERS from the most recent parse_dockerfile call" note above `covers_file`, the main-loop banner, the header's per-F/per-D algorithm prose) for stale per-pair phrasing the grep does not match, and list each straggler as a survey-derived correction in the change.
4. Tests (`assert-packaging-in-diff.test.sh`): a fixture Dockerfile with a backslash-continued multi-line `COPY` enumerating literal files — assert the floor flags an added sibling the continued `COPY` misses (exit 1) and passes one it covers (exit 0); a mid-continuation comment case; re-baseline any exact-output-order expectations disturbed by the inversion (pair set must be proven unchanged).

### B. Vacuous-packaging advisory (`assert-packaging-in-diff.sh` + `workflow-template.js`, #819)

1. New optional flag `--advise-vacuous` in the script's argument parser (style mirrors `--repo`). With the flag, the two structural no-op exits print one stderr line each before `exit 0`, e.g. `assert-packaging-in-diff: advisory — requiresPackaging declared but no Dockerfile at <branch>; floor is structurally vacuous (ratified scope, ADR 0017 addendum)` and the A/R/C-empty analogue. Without the flag, behavior is byte-identical to today. Stdout and all exit codes unchanged.
2. In `workflow-template.js`, append ` --advise-vacuous` to the `assert-packaging-in-diff.sh` invocation string in the dispatched refiner prompts **only when** `r.task.requiresPackaging === true` (strict), at every site the floor is instructed.
3. **Token sweep (grep is a floor):** grep `assert-packaging-in-diff.sh` across `skills/war/assets/workflow-template.js` and thread the conditional flag at every dispatched invocation site (initial merge, retry-merge, re-merge re-instruction; the polish-sweep and `requiresPackaging:false` skip clauses stay flagless) — then **hand-scan the same-scope prompt-building blocks plus `agents/war-refiner.md` and `agents/war-worker.md`** for prose describing the packaging floor's invocation or output without the literal script-name token (the standing-vs-dispatched split drifts silently), and list each straggler as a survey-derived correction.
4. ADR 0017 gains a short dated addendum note: the ratified scope now carries an explicit-declaration advisory surface; scope itself unchanged.
5. Tests: script-side (`assert-packaging-in-diff.test.sh`) — flag + no-Dockerfile fixture → exit 0 + advisory on stderr; flag + Modified-only diff → exit 0 + advisory; flag + genuinely flagged case → exit 1, no advisory; no flag → stderr silent on both vacuous shapes. Engine-side (`workflow-template.test.mjs`) — the built merge prompt contains `--advise-vacuous` iff the task's `requiresPackaging` is explicitly `true`, and omits it for a defaulted (undefined) task; delete-the-feature check: the assertion fails if the strict-equality condition is loosened to the `!== false` default (lesson `weak-test-assertion-passes-without-feature-being-exercised`).

### C. Ref-accurate `.gitmodules` read (`skills/war/assets/assert-no-submodule-mutation.sh`, #802)

1. Replace Step 3's working-tree resolution (the `gitmodules_file` existence check + `git config -f`) with: probe `$git_cmd cat-file -e "$branch:.gitmodules"` — on failure, skip Step 3 clean (no declared paths, today's no-op semantics); on success, `submod_paths="$($git_cmd config --blob "$branch:.gitmodules" --get-regexp '\.path$' | awk '{print $2}')"` with a non-zero pipeline dying `2` (git error, never a clean pass). The `repo_dir`-aware `$git_cmd` prefix already exists; the `gitmodules_file` variable is retired.
2. **Token sweep (grep is a floor):** grep `git config -f` and `gitmodules_file` across `assert-no-submodule-mutation.sh` and convert every working-tree read — then **hand-scan the script's header comment block and Step-3 banner** (the usage/mechanics lines citing "cross-checked via `git config -f .gitmodules`" and "Resolve .gitmodules location") **plus the test file's case titles and comments** for working-tree phrasing, and list each straggler as a survey-derived correction.
3. Tests (`assert-no-submodule-mutation.test.sh`): a fixture where the **branch itself adds/edits `.gitmodules`** declaring a new submodule path and touches non-gitlink content under it with `--declared` — assert refusal keys on the branch-ref declaration (would misclassify under a working-tree read whose checkout lags the branch); the inverse case (working tree carries a declaration the branch ref lacks) proving the working tree no longer leaks in; a ref-present-but-unreadable arrangement is not force-manufactured (documented as untestable-cheaply if so).

### D. `extract_msg` fidelity (`skills/war/assets/assert-guard-specificity-in-diff.sh`, #803)

1. In `handle_added_line`'s stderr branch, extract the message from the emit segment: locate the emitting keyword (`echo`/`printf`, the token set the header contract names) and call `extract_msg` on the text between it and the `>&2` token, instead of on the whole line. Lines with `>&2` but no recognizable emit keyword keep today's whole-line behavior (documented ceiling).
2. In `record_guard` (single choke point — both the `die` branch and the stderr branch route through it), add: a message that is a bare variable reference (`$name`/`${name}` only) → return without recording; a message containing a `%` conversion → truncate to the prefix before the first `%`, and if the trimmed prefix is empty → return without recording.
3. **Token sweep (grep is a floor):** grep `extract_msg` across `assert-guard-specificity-in-diff.sh` and apply the adjacency/bare-var rules at every caller — then **hand-scan the header's MSG contract comment (`MSG = the first quoted string literal …`) and the KNOWN LIMITS comment block** for statements of the old first-quote-on-line rule, updating them to the new contract in the same diff, and list each straggler as a survey-derived correction.
4. Out of bounds (constraint 3): `match_sh_suite`, `match_default`, and every discovery case arm are untouched — the sibling drift-guard-tightening plan owns parity locks on those.
5. Tests (`assert-guard-specificity-in-diff.test.sh`): shape-1 fixture (`[ -f "$path" ] || { echo "real msg" >&2; exit 1; }` with a test asserting `real msg`) → covered, not flagged; shape-1 negative (test asserts the old mis-extracted `$path` token but not the real message) → flagged; `printf 'error: %s' …>&2` guard with a test containing `error: ` → covered via the truncated prefix; shape-2 fixture (`die "$msg"`) → produces **no** uncovered line at all; a partially-interpolated literal still records and matches.

## 8. Open risks / implementation notes

- **Same-file cohesion for plan carving:** #800 and #819 both edit `assert-packaging-in-diff.sh` + its test — they must not be parallel tasks (code-boundary rule 1). #802 and #803 are file-disjoint from everything else.
- **Cross-plan shared-file contention:** `workflow-template.js`/`workflow-template.test.mjs` are also touched by the audit-gate-evidence-fidelity sibling (hence the build-after edge), and `assert-guard-specificity-in-diff.test.sh` gains gate-parity cases from the 2026-07-11 drift-guard-tightening plan — the roadmap's shared-file-contention table must carry both edges.
- **Loop inversion ordering:** the flagged-pair *set* is the contract; line order is not. Still, prove set-equality in the re-baselined tests rather than silently rewriting expectations (a reordered expectation that also drops a pair would pass a naive rewrite).
- **`git config --blob` fixture cost:** the C tests need commits where `.gitmodules` differs between working tree and branch ref — build fixtures with plumbing (`git update-index`/commit then dirty the working tree), not checkout gymnastics; keep them cwd-independent.
- **Advisory wording is load-bearing prose:** it must say *ratified scope*, citing the ADR 0017 addendum, so a reader routes to the docker gate/CI instead of filing the no-op as a bug again (that mis-read is this issue's own origin).
- **#803 emit-keyword set:** keep it to the tokens the header contract already names (`echo`, `printf`); widening to arbitrary commands before `>&2` re-opens the first-quote-anywhere problem in new clothes. `# ponytail:` the ceiling: multi-command emit segments and `cat <<EOF >&2` heredocs stay undetected, as today.
- **Do not let B's threading touch the polish-sweep exemption** — the coherence sweep's "skip the packaging floor" clause is deliberate and stays flagless and skip-worded.

## 9. Non-goals / deferred

- **No scope widening of the packaging floor** — Modified/Deleted paths stay unflagged (ADR 0017 addendum is not overturned); no JSON-array/heredoc `COPY` parsing (spec-§9 deferral stands); no `.dockerignore` subset expansion.
- **No promotion of the guard-specificity floor** toward blocking, no MergeResult status, no `HARD_ESCALATION_REASONS` change; the secondary-die-path Minor-grading behavior (lesson `guard-specificity-floor-flags-secondary-die-paths-as-routine-minor`) is out of scope.
- **No gate-set parity cases** in any of these test files — owned by the drift-guard-tightening plan.
- **No engine-side evidence field** for the vacuous advisory (no schema, no relay, no audit-lens wiring).
- **No base-ref or union `.gitmodules` read** in the submodule floor — branch-ref is the single declared-path source; the gitlink-mode check remains the ref-independent backstop.
- **No `--advise-vacuous` in the /war SKILL.md hand-run recipes** — the flag is an engine-threaded, explicit-declaration signal; hand-run recipes stay minimal.

## 10. Validation criteria

1. `bash skills/war/assets/assert-packaging-in-diff.test.sh` passes, including: a backslash-continued multi-line enumerated `COPY` fixture where the floor **flags** an added uncovered sibling (exit 1) and **passes** a covered one (exit 0) — both RED against the pre-change script (the continuation was previously dropped, fail-open).
2. A mid-continuation `#` comment fixture parses to the same source set as the comment-free equivalent.
3. `parse_dockerfile` is invoked exactly once per Dockerfile per run (assertable via a wrapper/trace in a test, or by construction: the call sits in the Dockerfile-outer loop with no other call site — grep proves single call site), and the flagged-pair **set** for a multi-F × multi-D fixture equals the pre-change set.
4. With `--advise-vacuous`: no-Dockerfile fixture → exit 0 + one stderr advisory naming the no-Dockerfile shape and citing ADR 0017; Modified-only-diff fixture → exit 0 + the A/R/C-empty advisory; a flagged run → exit 1 with unchanged stdout pairs and no advisory. Without the flag: stderr byte-empty on both vacuous shapes (today's behavior).
5. `node --test skills/war/assets/workflow-template.test.mjs` passes with new cases: the dispatched merge prompt carries `--advise-vacuous` for a task with explicit `requiresPackaging: true`, omits it for a defaulted task, and the case is proven load-bearing (mutating the condition to the `!== false` default turns it RED).
6. `bash skills/war/assets/assert-no-submodule-mutation.test.sh` passes, including the branch-adds/edits-`.gitmodules` `--declared` case keying on the branch-ref declaration, and the inverse case proving a working-tree-only declaration no longer influences Step 3; `grep -c 'git config -f' assert-no-submodule-mutation.sh` → 0.
7. Blob-read failure after a successful `cat-file -e` probe exits 2, never 0 (asserted or, if uncheaply manufacturable, recorded as a documented deferred validation in the plan's backstops section).
8. `bash skills/war/assets/assert-guard-specificity-in-diff.test.sh` passes, including: shape-1 fixture covered via the real message (RED pre-change: `$path` mis-extracted), shape-1 negative flagged, `printf` format-prefix match covered, shape-2 `die "$msg"` produces no uncovered line (RED pre-change: perpetual flag), partially-interpolated literal still recorded.
9. The three updated header/contract comment surfaces (packaging main-loop/`D_COVERS` notes, submodule Step-3/header notes, guard-specificity MSG contract + KNOWN LIMITS) contain no statement of the superseded mechanism — verified by the three manual same-scope surveys in §4 (A.3, C.2, D.3), with any straggler listed as a survey-derived correction in the landed diff.
