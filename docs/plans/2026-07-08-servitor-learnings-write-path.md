# Servitor learnings write path — local-root-anchored writes, Gate-2 promotion via a publication worktree, and a user-memory mutation guard

Source spec: [2026-07-08-servitor-learnings-write-path-design.md](../specs/2026-07-08-servitor-learnings-write-path-design.md)
Issues: #584 (learnings written off the working branch + destructive edits to hand-curated personal memory). Also resolves the ratified #58 deferral recorded in `hooks/validate-worktree-scope.sh`'s header (glob subtraction, not anchoring — see Task 2).

## Commander's Intent

- **Purpose:** a landed phase's lessons must actually ride the phase PR, and the operator's personal memory store must be immutable to agents. Today `commitLearnings` (default true) publishes nothing — the servitor's relative write target resolves against the launch session's cwd, off the working branch, and Gate 2 commits in a working-branch checkout that post-checkout-guard **never exists** — while the servitor's D1/D2 disciplines affirmatively sanction editing pre-existing hand-curated memory files.
- **Method:** convert the spec's resolved tree A–I without re-litigating it, with two operator-ratified deviations (copy-with-marker promotion; `p<N>-publication` naming). Collapse the servitor's write surface to **one root**: Setup threads a new `memoryLocalRoot` arg (absolute local root — the servitor's only writable path; `learningsTarget` is retained untouched as the read-path resolved repo root feeding `workerSelfQueryRepoFlag` and Gate-2 promotion). The Wrap-up dispatch gates on `memoryLocalRoot` (absent — the named producer is Setup's memory-probe failure — ⇒ skipped + logged, fail-open). Repo-root publication becomes a **Lead Gate-2 promotion**: reconcile `servitorResult.files_written` against the local root (absolute-prefix check; any stray path fails loud), select `type: project` files, provision a transient **`p<N>-publication`** worktree on the working branch via a new `provision-worktrees.sh` subcommand pair (mirroring the `ensure-refinery-worktree` idiom — single-tested-owner doctrine, never prose `git worktree add`), copy + redaction-lint there (fail-closed: a flagged file stays local, never dropped; a same-slug repo file is **overwritten on promote** — the ratified update mechanism for recurrence edits), commit `docs(learnings): phase N` + the byte-identical CLAUDE.md pointer duty, push via the existing `ensure-origin` CAS (any failure ⇒ fetch, replay the docs commit, retry once, then escalate — never force), remove the worktree, then **stamp each promoted local original with a nested `metadata.promoted:` marker** (promotion is a copy-with-marker, never a move or delete — the local row survives for in-run recall) before the final `render-index`. A leftover publication worktree from a crash is **healed** by a Setup pre-flight and Gate-2 entry scan (clean ⇒ remove; dirty ⇒ escalate). Tighten both hooks shape/content-based only (hooks cannot receive per-run values): the scope hook's servitor branch **drops** the `*/docs/learnings/*` alternative; `validate-servitor-provenance.sh` gains an **existing-target mutation guard** — a pre-existing memory file whose frontmatter lacks nested `metadata.provenance` is user-authored, top-of-ladder (ADR 0007), and immutable to the servitor (Write, Edit, and NotebookEdit all denied). Every dispatched-prompt change lands with its `agents/war-servitor.md` mirror in the same commit (both-surfaces rule). No enum change, no `land-decision.mjs` change, no `war-memory.mjs` change, no `SERVITOR_RESULT` schema change, no `/lessons-learned` change.
- **End state:**
  1. **Threading + Wrap-up gate** (`workflow-template.test.mjs`): with `memoryLocalRoot` set, the Wrap-up prompt contains the absolute local root and its **writable-path clause** does not name `docs/learnings` (scoped anchor — the prompt deliberately says "never write into any `docs/learnings/`" elsewhere, so a whole-prompt absence grep is wrong by construction); with `memoryLocalRoot` absent (even with `learningsTarget` present and the phase landed), **no servitor dispatch occurs** and a skip line is logged — both assertions fail if the gate or the clause is deleted.
  2. **Discipline prose, both surfaces:** the dispatched D1/D2 clauses and `agents/war-servitor.md`'s D1/D2/Routing/Never carry the provenance-presence qualifiers (untagged pre-existing file = user-authored, never edited — new cross-linked file instead) **and the repo-lesson recurrence flow** (covering lesson in the repo root ⇒ write the updated full copy into the local root, same slug, `type: project` — the local marked copy, when present, is the canonical recurrence-edit target and is provenance-tagged so the mutation guard allows it; Gate-2 promotion overwrites the same-slug repo file); both surfaces demand **absolute paths** in `ServitorResult.files_written`. A token-anchored, sentence-case-tolerant drift test proves both surfaces carry the mutation-guard tokens, the recurrence-flow token, and the absolute-path demand; that `agents/war-servitor.md` no longer carries the either/or routing tokens (the `phase-<N>.md` aggregate, the "else: append" arm); and that the template's args header comment no longer says "(memory dir or docs/learnings/)".
  3. **Scope hook** (`validate-worktree-scope.test.sh`): a servitor Write to any `*/docs/learnings/*` path is denied; a servitor Write under `*/.claude/projects/*/memory/*` is allowed; the `..` rejection and the worker/auditor branches are behavior-unchanged (every existing case still passes).
  4. **Mutation guard** (`validate-servitor-provenance.test.sh`): servitor Edit of an existing on-disk memory file without nested `metadata.provenance` → deny; Edit of an existing tagged file → allow; **decoy case (delete-the-feature): a servitor Write whose *content* carries a valid provenance tier over an existing *untagged* file → still deny** — the one case that fails only if the guard reads the disk, not the payload; Write to a new path with a valid tier → allow (regression); a top-level `provenance:` line in the target file does **not** count as tagged; a non-servitor Edit of the same untagged file → allow (back-compat); the `MEMORY.md` exact-basename exemption still passes.
  5. **Publication worktree** (`provision-worktrees.test.sh`): `ensure-publication-worktree` creates a worktree on the working branch at its tip; re-invocation reattaches idempotently (clean tree) and fails loud on a dirty one; `remove-publication-worktree` succeeds on a clean worktree and fails loud (non-zero + message) on a dirty one, never `--force`; after removal the working branch is checked out nowhere and its ref still resolves.
  6. **Gate-2 doc contract** (`war-config.test.mjs` locks): SKILL.md Setup step 4 names `memoryLocalRoot` (and names the memory-probe failure as the omission producer) and no longer carries the scope-hook-glob compatibility clause; Gate 2 names the `files_written` reconciliation assertion, the publication worktree, the retry-once CAS rule, the **copy-with-marker / marker-after-push** rule, and overwrite-on-promote; the **crash-heal clause** (Setup pre-flight + Gate-2 entry scan for a leftover `p*-publication` worktree) carries a lock token; the three existing Gate-2 locks (render-index `--local` **and** `--repo`, the pointer duty, prefetch `--repo`) stay green; the swept doc surfaces (`skills/war/SKILL.md`, `skills/war/references/design.md`, `skills/war/references/schemas.md`) no longer describe `docs/learnings` as servitor-writable.
  7. `CONTEXT.md` defines the three new terms (publication worktree, promotion — copy-with-marker semantics and the marker lifecycle, WAR-editable memory file); a new ADR records the write/publish split, the provenance-presence discriminator, the `metadata.promoted:` marker shape + lifecycle, the re-ratified #58 residual, and the accepted tagged-file mutability residual; `CLAUDE.md`'s ADR range and servitor guard sentence are updated.
  8. All four release slots carry the same new version; full gate green at the landing tip.

Rationale: a move creates a recall blind spot — `render-index`/prefetch/`/lessons-learned` read the launch checkout, where the promoted file is invisible until the campaign PR merges — and this repo has stranded-branch precedent (25 known-stranded remote branches). No new push subcommand.
