## AI-Commander's Intent

- **Method:** Retrofit the file's own `_tmp_err` mktemp idiom (already used in `cmd_ensure_integration`'s fetch and both branch-create sites) around `cmd_ensure_origin`'s push — keep the never-force guidance text, append captured stderr, leave the push command/refspec/`-u` byte-identical (ADR 0004); rework D9 pattern 1 to capture an optional dotted receiver anchored on a non-dotted boundary (`(?<![\w$.])`) and drop hits only when the captured receiver is in a new `NARRATION_RECEIVERS` allowlist (`mr`, `mergeResult`) beside `detectEnumLeaks` — chained paths (`result.mr.status`) never match the allowlist and still flag, the equality match is never loosened, patterns 2/3 untouched; add case **T2.9** whose bare-origin `pre-receive` hook exits non-zero, so `ls-remote` passes, the push fails with `! [remote rejected] … (pre-receive hook declined)` (no contiguous `[rejected]` token), and classification provably falls through to the bare `exit 3` per the canonical 0/2/3 contract in `provision-worktrees.sh`'s `cmd_land_advance` header — with route-pinning evidence (a direct-push probe asserting the token fact, plus negative die-text assertions excluding the rc-guard and phantom routes), because exit 3 alone is shared by three other routes. Every comment naming pre-fix behavior is updated in the same commit as its behavior change (three grep sweeps, each with a mandatory manual same-scope survey — grep is a floor, not a ceiling; sweep 3 is report-only, it edits nothing outside the task's declared files).
