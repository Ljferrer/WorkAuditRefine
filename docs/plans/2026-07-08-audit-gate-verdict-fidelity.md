# Audit/gate verdict fidelity — implementation plan

**Source spec:** `docs/specs/2026-07-08-audit-gate-verdict-fidelity-design.md`
**Slug:** `audit-gate-verdict-fidelity` (shares the spec's slug, drops `-design`).
**Repo version at authoring:** 0.14.14 — version literals below are non-authoritative; resolve the next free patch from the four release slots at land time.

## Commander's Intent

**Purpose.** Move every deterministic part of the audit/gate anti-cheat spine — the pin/ancestor proof, the seat-tip equality check, the full-gate-output capture, the verdict-vs-severity hard path — out of prose the agent may skip and into Node checks, refiner-run shell floors, and captured artifacts, so no seat ever hand-reconstructs a git proof and no finding-less `escalate` silently lands.

**Method.** Add two new refiner-side floor scripts (`gate-pin-status.sh` computes the CONFIRMED / BENIGN-ADVANCE / STALE-MISMATCH proof; `assert-guard-specificity-in-diff.sh` catches a new `die`/stderr guard with no same-diff test asserting its message) with the standard 0/1/2 exit discipline and `--mapped`/pattern defaults mirroring `resolveGate`. In the Workflow: a pure-Node pin-equality gate demotes any seat whose returned `audit_sha` mismatches its dispatched pin to `pin-mismatch`/SOFT; `isHardGateEvidence` (and the end-state seat's `isHard`) also fires on `verdict === 'escalate'` — reusing the existing `gate-evidence` reason, **no** `HARD_ESCALATION_REASONS` change (ADR 0005). A single consolidated post-merge refiner "evidence dispatch" stamps per-task `pin_status` + guard-specificity evidence and, on an intra-phase-deps phase, re-runs the gate once at the final integration tip as the land-authoritative execution evidence. The refiner tees full gate stdout+stderr to a `.war/gate-<taskId>.log` artifact and returns its absolute path in `MergeResult.gate_log_path`, structurally replacing the *"Do NOT curate or excerpt"* prose. Every dispatched-prompt clause lands in the same commit as its `agents/*.md` mirror. All evidence plumbing fails open — a missing token/artifact is a SOFT cannot-confirm, never a hold.

**End state** (each individually checkable):
1. `skills/war/assets/gate-pin-status.sh` exists; `bash gate-pin-status.test.sh` is green covering all four exits — CONFIRMED (equal SHAs, exit 0), BENIGN-ADVANCE (descendant + none of **the task's own changed files** (the explicit `--mapped` list the pipeline passes) touched in the intervening range, exit 0, intervening file list printed), STALE-MISMATCH (a task-own file changed or not an ancestor, exit 1, offending files printed), git/ref error or the `(integration_sha …)` sentinel → exit 2 (never collapsed into a status). (spec criterion 1)
2. `grep -n 'gate-pin-status' agents/war-refiner.md skills/war/assets/workflow-template.js` shows the refiner invocation and the seat-prompt token consumption; the gate-audit seat prompt no longer instructs the auditor to hand-run the `cat-file` → `rev-parse`-compare → mapped-diff recipe (replaced by "consume the stamped `pin_status`"); STALE-MISMATCH / exit-2 preserve the stale-tip SOFT-defusing rule. (spec criterion 2)

## Build order (for /war)

### Phase 1 — Floors, hard-path semantics, doctrine docs

Four file-disjoint tasks, no intra-phase deps — run in parallel.

**Task 1.1 — `gate-pin-status.sh` floor + test (D1)**
- Files: `skills/war/assets/gate-pin-status.sh` (new), `skills/war/assets/gate-pin-status.test.sh` (new)
- Plan slice: New floor script, sibling of `assert-test-in-diff.sh` — same bash-3.2-safe, cwd-independent style, same `die`/exit discipline, same `--repo <git-dir>` test-only flag. Usage: `gate-pin-status.sh <gateHeadSha> <observedHead> [--mapped <file-list>] [--repo <git-dir>]`. `--mapped` is the **explicit per-task file list** the pipeline always passes (Task 2.1 computes it as that task's own changed-file set); the mapped set is deliberately NOT the global gate-discovery set — sibling merges in the serial queue nearly always add their own test files, so a global test-glob default would read every non-final task as STALE-MISMATCH and permanently defuse the provably-unrun HARD path. A global default (the `assert-test-in-diff.sh` discovery set) may remain for standalone/manual invocation only and MUST be documented in-script as NOT what the pipeline passes. Contract: (a) `git cat-file -t <gateHeadSha>` fails **or** `<gateHeadSha>` is the `(integration_sha …)` sentinel shape ⇒ exit 2 (git/ref error — never a status; caller treats as cannot-confirm SOFT); (b) `gateHeadSha == observedHead` ⇒ exit 0, print `CONFIRMED`; (c) `merge-base --is-ancestor <gateHeadSha> <observedHead>` holds AND `git diff --name-only <gateHeadSha>..<observedHead>` intersected with the `--mapped` set is empty ⇒ exit 0, print `BENIGN-ADVANCE` + the intervening file list (cited evidence); (d) ancestor holds but a mapped file changed, or not an ancestor ⇒ exit 1, print `STALE-MISMATCH` + the offending files. (No `floor ⊆ gate` mirror claim here — this floor classifies pin drift, it does not admit test evidence; the `assert-test-in-diff.sh` default is a different category, not "the same".) Test covers all four exits in a fixture repo (the `assert-test-in-diff.test.sh` scaffold idiom), including the sentinel-input → exit 2 case, and delete-and-traces its own assertions (each case fails if its branch of the contract is removed).

**Task 1.2 — `assert-guard-specificity-in-diff.sh` floor + test (D6 mechanical arm)**
- Files: `skills/war/assets/assert-guard-specificity-in-diff.sh` (new), `skills/war/assets/assert-guard-specificity-in-diff.test.sh` (new)
- Plan slice: New floor modelled on `assert-test-in-diff.sh` (same style, exits, `--repo`, optional `--pattern <glob-set>` unioned like the test floor). Compute the three-dot `--name-only`/`-U0` diff `<base>...<branch>`; in **added** lines of non-test files, detect new early-exit guards carrying a quoted message emitted to stderr (`die "…"`, `printf/echo "…" >&2` adjacent to an `exit`/`return` — a line-based shell heuristic, documented ceiling per the Dockerfile-parser precedent, `# ponytail:` comment naming the ceiling and upgrade path). For each unique new message literal, require that some **test file in the same diff** (match set = the `assert-test-in-diff.sh` gate-mirror default unioned with `--pattern`, preserving floor ⊆ gate) contains that substring in its added lines. Exit 0 = no new guard or all covered; exit 1 = at least one uncovered guard message — print the message + defining file (the auditor's cited evidence); exit 2 = git/ref error, never collapsed into exit 1. This floor is **advisory-evidence**: it never mints a MergeResult status (no `HARD_ESCALATION_REASONS`/enum widening — ADR 0005; wiring is Task 2.1). Test: fixture-repo cases for uncovered-guard ⇒ 1 with the message printed, covered-guard ⇒ 0, no-new-guard ⇒ 0, non-shell change ⇒ 0, bad ref ⇒ 2; delete-and-trace each.
