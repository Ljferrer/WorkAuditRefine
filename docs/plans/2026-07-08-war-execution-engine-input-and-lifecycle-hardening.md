## Commander's Intent

**Purpose.** WAR's engine seams stop converting imperfect input or half-run state into raw `TypeError`s, silent no-ops, or dangerous manual recovery ‚Äî every trust boundary returns a named clean error, mechanically, and the provisioning lifecycle heals its own provably-safe orphans instead of demanding hand-edited git refs.

**Method.** **(Band 1 ‚Äî ingest guards)** `war-config.mjs` `validate()` gains the `overrides` non-object guard byte-mirroring the sibling `memory` block idiom. Both sandbox args-parse sites gain a scalar-safe non-null-object guard ‚Äî hand-mirrored because the Workflow sandbox cannot import, pinned by a drift-guard test: the template **throws** a named error routing to the existing `held:workflow-error` (never a new enum member ‚Äî ADR 0005); the scaffold falls back to `{}` so the existing titleLine refusal fires cleanly. A `pt` **tagged prompt template** (Option B ‚Äî operator-ratified 2026-07-10 after this plan's first `/war` audit disproved the whole-prompt `\bundefined\b` scan) checks each interpolated **value** for identity `=== undefined` at prompt-build time ‚Äî **zero false positives by construction** (text is never scanned, so quoted prose "undefined" in finding titles/test names/code can never trip it); every prompt-rendering template literal is tagged (~20 spawn sites, awaited or not ‚Äî incl. the non-awaited `runSeat` ‚Äî plus prompt-builder helpers like `auditPrompt`), optional fields carry explicit `?? '<unset>'` defaults, and a missing input throws before spawn naming the adjacent literal fragment ‚Äî generalizing the #586 derivation-path fix to every interpolated field. No `dispatch()` wrapper; `dispatchKind` rides plain spawn `opts`. **(Band 2 ‚Äî provisioning lifecycle)** `inject-campaign-state.sh` replaces the space-splitting `xargs ls -t` with an indexed-array + process-substitution line-read (bash-3.2-safe; fail-open posture untouched). `provision-worktrees.sh` gains the `readonly EX_*` exit-code catalogue (every coded `die` uses a constant; a test forbids uncatalogued numerics; the surfacing contract stays "any non-zero = halt"), an explicit optional target-repo arg on `ensure-exclude`, and the opt-in `--reclaim-empty-orphan` two-proof self-heal on `ensure-integration` (empty `git log <base>..<branch>` AND absent from origin ‚áí reclaim; either proof fails ‚áí the unchanged exit-3 die ‚Äî ADR 0003 fail-loud default and ADR 0008 never-destroy-work honored). **(Band 3 ‚Äî dispatch ergonomics)** Stable `opts.dispatchKind` discriminators on the provisioning dispatches (and the evidence dispatch plan 2 added) so mocks/handlers/audits stop parsing label prefixes; the `ponytail:`/`deliberately-unwired:` marker becomes a recognized auditor dead-code exemption. ADR 0034 + an ADR 0013 addendum ratify.

**End state** (each individually checkable):
1. `validate({overrides: null})` and `validate({overrides: 'x'})` return `{ valid: false, errors: [...] }` containing "overrides must be an object" and do not throw; a valid overrides object validates as before; reverting the guard makes the null case throw (delete-the-feature). (spec criterion 1)
2. Invoking the template Workflow with `args = 'null'`/`'true'`/`'5'` yields `held:workflow-error` naming "args must be a JSON object" ‚Äî not a raw `TypeError` ‚Äî with zero agents dispatched (the catch renders `phase: null` via the hoisted `phaseId` fallback, never a secondary `TypeError` on an unassigned `ph`); the scaffold normalizes the same scalars to `{}` and produces the clean titleLine refusal; a drift-guard asserts the guard at **both** parse sites. (spec criterion 2)
3. A `pt`-tagged prompt template interpolating an `undefined` value throws at build time (before spawn) naming the adjacent literal fragment; a **defined** value whose text contains the word "undefined" does NOT throw (the zero-false-positive control); reverting the tag (or untagging a spawn-site literal) lets the prompt ship (delete-the-feature). (spec criterion 3, revised 2026-07-10)
4. With a campaign directory path containing a space, `inject-campaign-state.sh` still selects and injects the newest active campaign (the `xargs` version fails this); no-campaign ‚áí silent exit 0; newest-first order preserved. (spec criterion 4)
5. Every `die "‚Ä¶" <n>` in `provision-worktrees.sh` uses a catalogued `EX_*` constant (grep assertion ‚Äî no uncatalogued numeric literal); the catalogue names codes 3/4/5/6/7 with their governing ADRs; the "any non-zero = halt" surfacing contract is documented and asserted. (spec criterion 5)
6. `ensure-exclude <repo-dir>` invoked from a different cwd writes the exclude into `<repo-dir>`'s git dir; the no-arg form is byte-identical to today. (spec criterion 6)
7. With `--reclaim-empty-orphan`, a same-namespace unowned branch proven empty AND origin-absent is deleted and re-cut; the same branch with one unique commit ‚áí exit `EX_FOREIGN` (3), not deleted; present on origin ‚áí exit 3, not deleted; flag absent ‚áí exit 3, byte-identical default. (spec criterion 7)
8. The phase git-topology barrier and the per-task provision-run carry **distinct** `opts.dispatchKind` values; the test mock, `isProvision`, and `isProvisionRun` key on `dispatchKind` with no label-prefix regex; collapsing the kinds fails the distinctness assert (delete-the-feature). (spec criterion 8)
9. `agents/war-auditor.md` names the `ponytail:`/`deliberately-unwired:` marker as a dead-code-finding exemption (presence check, case-tolerant mid-sentence anchor). (spec criterion 9)
10. `agents/war-refiner.md` reflects the explicit-target `ensure-exclude` call; the `dispatchKind` names in template prompts and the refiner modes in `agents/war-refiner.md` are consistent ‚Äî no orphaned discriminator. (spec criterion 10)
11. `docs/adr/0034-*.md` (engine ingest guards & provision exit-code contract) exists; the ADR 0013 addendum records the deliberately-unwired marker; `CONTEXT.md` carries the six new terms. (spec ¬ß6/¬ß7; renumbered ‚Äî see deltas)
