# Land-path integrity and status-enum discipline — implementation plan

**Source spec:** `docs/specs/2026-07-08-land-path-integrity-and-status-enum-discipline-design.md`
**Slug:** `land-path-integrity-and-status-enum-discipline` (shares the spec's slug, drops `-design`).
**Repo version at authoring:** 0.14.14 — version literals below are non-authoritative; resolve the next free patch from the four release slots at land time.

## Commander's Intent

**Purpose.** Make a WAR phase land only when git proves the working ref advanced — no phantom lands, no ref lag, no manual-land protocol the Lead has to remember — and convert the two prose-only status-enum invariants into drift-guarded tests, all without adding a single new status, `HARD_ESCALATION_REASONS`, or `KNOWN_LAND_DECISIONS` member.

**Method.** Route every land (in-flow, auto-recover, escalation-completion) through the one primitive `cmd_land_advance`, and add a phantom-land guard there anchored on **git ground truth — the pre-push origin tip** (`git ls-remote origin refs/heads/<working>` captured inside the subcommand immediately before the push), never the local follower `refs/heads/<working>` (which this plan's own thesis says lags): refuse with exit 3 when `new_sha` equals the pre-push origin tip **and** the local follower already sits at it (proof the merge produced nothing new); reconcile-and-exit-0 when origin is already at `new_sha` but the follower lags (an interrupted prior land — ADR 0008 repair toward git); skip only on an **empty origin readback** (true first land). Teach the reland loop to tell a contender-less transient from a real divergence via `git rev-list --left-right --count <merge-sha>...origin/<working>` after a fresh fetch — right count 0 buys exactly **one** extra push-first attempt beyond `roundLimit`, mirrored byte-parallel in the standing `agents/war-refiner.md` §land-phase and the dispatched `workflow-template.js` land prompt in one task. Rewrite the SKILL.md `held:land-failed` auto-recover to branch on root cause — checkout collision (existing), and absent-origin via empty `ls-remote` → `ensure-origin`, the latter gated on a `merge-base --is-ancestor` trustworthiness predicate before origin is bootstrapped from a local ref — and collapse the escalation-completion dance onto the same detach-at-origin + `merge --no-ff` + `land-advance` topology every land uses. Pin the per-mode reachability of `HARD_ESCALATION_REASONS` (against the exported constant + the emitted land prompts, keeping the hand-written unreachability comment as narration) and the phase/task enum separation with tests. Ratify the doctrine in a new ADR.

**End state** (each individually checkable):
1. `land-advance` exits **3** (via `die "<msg>" 3` — `die()` takes an exit-code second arg) with the phantom-land message and leaves `refs/heads/<working>` and origin unchanged when `<new-sha>` equals the **pre-push origin tip** and the local follower already sits at it; when origin is already at `<new-sha>` but the follower lags or is absent it exits **0** as *already landed*, reconciling the follower toward git (no push); a genuinely-advanced `<new-sha>` still exits 0 and advances the follower; an **empty pre-push origin readback** (true first land) exits 0 with the guard skipped and the post-push readback still enforced; a *failed* `ls-remote` (non-zero rc) exits 3 — a git error never collapses into the first-land carve-out. (spec criteria 1, 2; operator decision 2)
2. On final CAS-attempt failure the reland loop fetches and runs `git rev-list --left-right --count <merge-sha>...origin/<working>` (the merge sha it tried to push vs. the fresh origin tip — never the lagging local follower): right count **0** ⇒ contender-less transient ⇒ exactly **one** extra push-first land attempt beyond `roundLimit` exhaustion, then `land_stale`; a nonzero right count ⇒ real contender commits ⇒ `status:"land_stale"` immediately — the identical discrimination, including the explicit-+1 budget, present in **both** `agents/war-refiner.md` §land-phase and the `workflow-template.js` land prompt (grep parity). (spec criteria 3, 10; operator decision 2d)
3. A land that resolves after a transient returns `status:'landed'` and reaches the existing `servitorResult` gate (`landResult.status === 'landed' && memoryLocalRoot`) with no Lead intervention — `servitorResult` non-null. (spec criterion 4)
4. The SKILL.md Checkpoint `held:land-failed` auto-recover branches on **both** root causes — checkout collision (existing), and absent-origin baseline detected via empty `git ls-remote origin refs/heads/<working>` → `ensure-origin`, the latter auto-firing under `--afk` **only** when `git merge-base --is-ancestor <working> <integration>` holds (never bootstrap origin from an untrusted local ref) — both ending in `land-advance` on a green gate; concrete escalation-completion prose is **authored** in SKILL.md **and** the memory lesson `held-escalation-lead-manual-completion.md`'s raw 6-step recipe is **updated** to the same one-primitive topology as every land (detach `_refinery` at `origin/<working>`, `merge --no-ff`, resolved gate, single `land-advance <working> <merge-sha>`), with **no** raw `git push`/`update-ref`/`ls-remote`/`--force-with-lease` step and **no** rebase on either surface. (spec criteria 5, 6; operator decision 3; grill Q6; red-team D7 re-anchor)
5. A drift-guarded test pins the land-phase-reachable subset of `HARD_ESCALATION_REASONS` against the **exported constant + the emitted land-phase prompts** and fails if `no-test`/`unpackaged` becomes reachable from any land-phase prompt; the hand-written unreachability comment at the `.includes(landResult.status)` site **stays** as narration. (spec criterion 7, amended per grill Q7)
6. A doc-contract test sources the task-status enum **and** the `landDecision` enum fresh from `schemas.md` and fails on a phase-level `landDecision` token used in a task-level status-equality context; it passes on the current tree and does not false-trip on `war-worker.md`'s "landed SHA" narration. (spec criterion 8)
7. `HARD_ESCALATION_REASONS` and `KNOWN_LAND_DECISIONS` are byte-unchanged across both mirrors and all four doc surfaces; the existing drift-guards still pass (`KNOWN_LAND_DECISIONS` doc-surface parity in `land-decision.test.mjs`; the `HARD_ESCALATION_REASONS` inline-mirror in `war-config.test.mjs` — verification must run both files, not just `land-decision.test.mjs`). (spec criterion 9)
8. A new ADR ratifies "the land asserts git ground truth; a `landed` result is never self-reported"; `CONTEXT.md` carries the four new terms; `node --test 'skills/**/*.test.mjs'`, every `hooks`+`skills` `*.test.sh`, and `war-memory lint docs/learnings/` pass. (spec criterion 11)
9. The four release slots (`plugin.json` `version`, `marketplace.json` `metadata.version` **and** `plugins[0].version`, `README.md` `## Status`) move together to the resolved next patch.

## Build order (for /war)

### Phase 1 — Land-truth guard, reland-loop discrimination, operator-facing prose

Three file-disjoint tasks, no intra-phase deps — run in parallel.

**Task 1.1 — Phantom-land guard in the land primitive (D1), origin-anchored**
- Files: `skills/war/assets/provision-worktrees.sh`, `skills/war/assets/provision-worktrees.test.sh`
- Plan slice: In `cmd_land_advance`, **before the push**, capture the pre-push origin tip: `pre_push_origin="$(git ls-remote origin "refs/heads/$working" | cut -f1)"`, checking the `ls-remote` exit code — non-zero rc (network/remote error) ⇒ exit 3 immediately; a failed readback must never collapse into the empty/first-land reading. The operand is a **new capture inside the subcommand, not a new CLI arg**: the 2-arg `land-advance <working> <new-sha>` contract stays stable for every caller (both workflow land prompts, SKILL.md D6/D7 prose, the submodule 2A path), and an in-subcommand capture happens at the last moment before the push (smallest race window) instead of going stale in caller prose. Then branch:
  - `pre_push_origin` **empty** (rc 0, no ref — true first land): guard skipped; the push creates the branch; the existing post-push readback still enforces `actual == new_sha`. This re-keys the first-land carve-out from the old empty-local-ref reading to an **empty origin readback** — the same signal D6's absent-origin detector reads (Task 1.3; see deviations for the two-consumer interplay).
  - `pre_push_origin == new_sha` **and** the local follower `refs/heads/<working>` also equals `new_sha` ⇒ **phantom land** (escalate, exit 3): `new_sha` equals the pre-push origin tip and the follower is not behind it, so this land did not advance origin. **Provenance correction (red-team):** the follower is *not* advanced solely inside `land-advance` — `cmd_ensure_integration`'s behind-case fast-forwards `refs/heads/<working>` toward the origin tip via `git update-ref` (the `do_follower_ff` branch) at the next phase's Provision. So `follower == new_sha` has **two** origins: a genuine phantom (the `--no-ff` merge produced no commit — integration had nothing ahead of `<working>`), **and** the rare **cross-resume already-landed** case where a prior interrupted land pushed `new_sha` and `ensure-integration` then ff'd the follower to it. **Both classify phantom → exit 3** — a deliberate, **fail-safe** collapse: escalating a benign already-landed is harmless (the work is already on origin; the operator confirms via the escalation-completion path) and never masks a real phantom nor reports a land that did not advance. It does **not** touch the load-bearing exit-0 reconciliation below. `die` with the phantom-land message (spec §4 wording, s/pre-push working tip/pre-push origin tip/) and **explicit exit 3**: `die "<msg>" 3` — `die()` is `printf … >&2; exit "${2:-1}"`, so the code arg is required or it exits 1 (grill Q4). All refs untouched; escalate class, never reland.
  - `pre_push_origin == new_sha`, follower **lags or is absent** ⇒ **already landed** (an interrupted prior attempt pushed `new_sha` but died before the follower CAS — ADR 0008 reconciliation): skip the push (it would no-op), CAS/create the follower to `new_sha`, exit **0**. This is the **in-loop transient-recovery path of Task 1.2**, where `ensure-integration` has **not** yet run to ff the follower, so the follower genuinely lags — the discriminator holds. (A *cross-resume* already-landed whose follower was ff'd to `new_sha` takes the phantom exit-3 path above — the fail-safe residual noted in Notes.) Repair the record toward git, never git toward the record; this is what makes an in-loop re-land idempotent instead of a false phantom.
  - `pre_push_origin != new_sha` ⇒ normal path, byte-unchanged: push, `[rejected]` classification, readback, follower CAS. The 0/2/3 exit contract is unchanged.
  The local follower is consulted **only as the phantom vs. already-landed tiebreak** — the guard's anchor is the origin tip (operator decision 2; the follower's lag is precisely the already-landed signature). Side payoff to assert in a test comment: the D6 checkout-collision recovery merges in the Lead worktree, where the merge advances the checked-out `refs/heads/<working>` to `<merge-sha>` *before* `land-advance` runs — local anchoring would false-phantom that; origin anchoring is what lets D6/D7 route through the primitive at all.
  `provision-worktrees.test.sh` cases (extend the existing T2 clone scaffold, don't duplicate it): (1) phantom — origin and follower both at `<tip>`, `land-advance <working> <tip>` → **assert exit code 3**, the die message, `refs/heads/<working>` and origin unchanged; (2) genuine advance → exit 0, follower advanced (extend the existing "working branch actually advanced" assertion); (3) first land — no `refs/heads/<fresh-working>` on origin → exit 0, guard skipped, post-push readback enforced; (4) already-landed — origin at `<new-sha>`, follower one commit behind → exit 0, follower reconciled to `<new-sha>`; (5) `ls-remote` failure — origin URL pointed at a nonexistent path → exit 3, never the carve-out.

- requiresPackaging: false
- deps: none

**Task 1.2 — Reland-loop transient-vs-divergence discrimination (D4/D5), both surfaces**
- Files: `skills/war/assets/workflow-template.js`, `agents/war-refiner.md`, `skills/war/assets/workflow-template.test.mjs`
- Plan slice: In the `agents/war-refiner.md` §land-phase push-first CAS loop (steps 2–3, superproject **and** submodule-2A variants) **and** the `workflow-template.js` land dispatch prompt (the `landDecision === 'landed'` block — both the in-flow land prompt and the baseline-proceed re-land prompt's reland sentence), add a pre-surrender step: on the final failed CAS attempt run `git fetch origin <working>` then `git rev-list --left-right --count <merge-sha>...origin/<working>` — the **merge sha the loop just tried to push** vs. the **freshly-fetched origin tip**, never the local follower `refs/heads/<working>` (it lags; operator decision 2d). **Contender-less transient, defined mechanically:** right count **0** — every commit on the fetched origin tip is already contained in the merge sha, so no competing commit exists and the rejection cannot be a lost CAS. That buys **exactly one extra push-first attempt beyond `roundLimit` exhaustion** (explicit +1, once — not a slot inside `roundLimit`): re-fetch, re-detach at `origin/<working>`, re-merge `--no-ff`, re-gate, `land-advance`; if that attempt also fails, return `status:"land_stale"`. A nonzero **right** count (real contender commits) ⇒ return `status:"land_stale"` with no extra attempt. The retry branch is live, not dead code: right=0 is reachable when the prior push actually landed but the client saw a transient error (the retry's `land-advance` then exits 0 via Task 1.1's already-landed reconciliation — left=0/right=0), or when a remote hiccup produced a spurious rejection (the recorded `phase-land-stale-spurious-cas-recovery` occurrences). **No new status or enum** — a transient that resolves returns `landed`, so the existing `servitorResult` gate spawns the servitor automatically; only a true divergence reaches the `HARD_ESCALATION_REASONS.includes('land_stale')` → `held:escalation` path. The two surfaces must state the identical discrimination **and the identical +1 budget sentence** and land in this one task (mirror-drift hazard — spec §8, grill Q5/Q11). Add `workflow-template.test.mjs` assertions (mock-agent harness, capture the emitted land prompt): the prompt contains the `rev-list --left-right --count <merge-sha>...origin/<working>` discrimination, the explicit-+1-then-`land_stale` budget, and `land_stale` only on a nonzero right count; grep parity of the same strings in `agents/war-refiner.md`; and a stale-then-resolved land (final `status:'landed'`) reaches the `servitorResult` dispatch with `servitorResult` non-null.
