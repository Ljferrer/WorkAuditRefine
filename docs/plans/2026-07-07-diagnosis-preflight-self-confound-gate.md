## Commander's Intent

- **Purpose:** WAR institutionalizes adversarial verification of what agents *build* but nothing of what they *conclude about failures* — #575's agent tore down its own cluster via back-to-back manual + automated bring-up, blamed a systemic coordination bug, and wrote a durable memory plus a fix-plan fan-out before reading the logs that proved there was no bug. Give every diagnosing role (red-team Lead, adversarial-confirm agents, WAR Lead, workers — including fix-workers, which are `war-worker`-seated and read the same standing file) the existing refute discipline aimed **inward**: no failure is attributed to a systemic bug, and no hypothesis is promoted into a durable artifact, until the agent has ruled itself out as the cause. The gate blocks **promotion, never diagnosis or escalation**.
- **Method:** prose-only standing instructions — no gate, floor, hook, `workflow-template.js`, or `war-config.mjs` changes (the group boundary with the sibling spec, whose mechanisms are code). The canonical four-part gate — action-provenance first, single-path validation, hypothesis promotion gated on primary evidence + an inward refute pass, state the falsifier — lives as one new section in `skills/red-team/SKILL.md`, closing with the escape valve (unobtainable evidence ⇒ labeled hypothesis note + escalate, then proceed). Condensed, context-tailored, **self-contained** clauses (never verbatim mirrors — lesson `verbatim-mirror-directive-context-mismatch-at-destination`) carry it to the surfaces that actually reach each role: `references/lenses.md` `## Safety` (Lead-facing reference doctrine), **one sentence inside `workflow-scaffold.js`'s string-built `confirmStage` prompt** (the only surface the runtime confirm agents ever see — a prose-content string edit, not machinery), `skills/war/SKILL.md` `## Invariants (never violate)`, `agents/war-worker.md` `## Stop and escalate instead of guessing`, and **`agents/war-servitor.md`'s verify-duty block** (operator-ratified: the write site of the incident's worst artifact) — each sharing the literal greppable term **"self-confound"**. One bash-3.2-safe text-scan test asserts presence per surface **and per canonical part** (presence, not byte-identity — non-identical by design). Hypothesis-promotion channels are a **closed list of four**: a memory/lesson write, a `war-followup` issue, a fix plan or spec, a sub-agent fan-out. An ADR records the doctrine and mirroring policy; CONTEXT.md ratifies the two terms.

The four numbered parts (spec §4): (1) **Action-provenance first** — before attributing any failure (a red probe, a broken baseline, an unexpected sandbox or live-system state) to the plan, the repo, or a subsystem, enumerate your own and any concurrent actor's recent **mutating** actions against the observed state and explicitly rule each out; "Did I cause this?" is question #1. (2) **Single-path validation** — never validate/reproduce against the same shared mutable state through two paths back-to-back (manual **and** automated); pick one path, re-provision fresh state before switching. (3) **Hypothesis promotion gated on primary evidence** — no durable artifact (a memory/lesson write, a `war-followup` issue, a fix plan or spec, a sub-agent fan-out — the closed D3 list) may encode a root-cause diagnosis until the mechanism is demonstrated from primary evidence (raw logs, a clean repro) **and** an inward refute pass has proven the cause is not your own action. (4) **State the falsifier** — before acting on a diagnosis, state the observation that would falsify it and go check; evidence merely consistent with the theory is not proof. Closing **escape-valve** sentence: the gate blocks *promotion*, never diagnosis or escalation — when primary evidence is unobtainable (ephemeral state, no logs), record the diagnosis as an explicitly-labeled **hypothesis** (report/ledger note) and/or escalate, then proceed. Cite the in-repo precedent: lesson `redteam-executed-probe-cwd-reset-hits-real-remote` is a recorded self-confound. Frontmatter `description:` untouched.
